## Goal

Build the `alma init --open-brain` quickstart command that takes a user from zero to a working personal brain in under 45 minutes. The command interactively configures storage, generates a valid ALMA config YAML, produces SQL migration scripts for PostgreSQL users, generates `.mcp.json` for Claude Code integration, and validates the entire setup with a test capture-and-retrieve cycle.

- `alma/cli/init.py` -- CLI command implementation

- `alma/cli/templates/config.yaml` -- Config template with smart defaults

- `alma/cli/templates/mcp.json` -- MCP config template for Claude Code
- `tests/unit/test_cli_init.py` -- Unit tests for CLI flow

## Steps

### 1. Design Interactive Init Flow

The `alma init --open-brain` command follows progressive disclosure -- ask only what is needed, provide smart defaults for everything else.

**Flow:**

```
$ alma init --open-brain

  Welcome to ALMA Open Brain Setup

  This will configure a personal brain that works with any MCP-compatible AI tool.

  Step 1 of 4: Choose Storage Backend

  [1] SQLite + FAISS (local, zero setup, single-tool use)
  [2] PostgreSQL + pgvector (recommended for multi-tool, Supabase-ready)
  [3] File-based (simplest, no dependencies)

  Your choice [2]: _

  Step 2 of 4: Configure Connection

  Database URL [postgresql://localhost:5432/alma]: _

  Or use Supabase? (y/n) [n]: y
  Supabase project URL: _
  Supabase service key: _

  Step 3 of 4: Embedding Provider

  [1] OpenAI (text-embedding-3-small, recommended)
  [2] Local (sentence-transformers, no API key needed)
  [3] Mock (for testing only)

  Your choice [1]: _
  OpenAI API key: _

  Step 4 of 4: MCP Configuration

  Generate .mcp.json for Claude Code? (y/n) [y]: _

  Done! Files created:
    .alma/config.yaml      -- ALMA configuration
    .alma/migration.sql    -- Database migration (run this on your PostgreSQL)
    .mcp.json              -- MCP configuration for Claude Code

  Next steps:
    1. Run the migration: psql $DATABASE_URL -f .alma/migration.sql
    2. Restart Claude Code to pick up .mcp.json
    3. Try: "Remember that I prefer TypeScript over JavaScript"
```

**Non-interactive mode** for CI/scripting:

```bash
alma init --open-brain \
  --storage postgresql \
  --database-url "postgresql://localhost:5432/alma" \
  --embedding openai \
  --openai-key "$OPENAI_API_KEY" \
  --mcp-json
```

### 2. Generate Valid ALMA Config YAML

The generated config must be loadable by `alma.config.loader.ConfigLoader`:

```yaml
alma:
  project_id: "open-brain"
  domain: "personal_brain"

  storage:
    backend: "postgresql"
    connection_string: "${ALMA_DATABASE_URL}"
    pool_size: 5

  embedding:
    provider: "openai"
    model: "text-embedding-3-small"
    api_key: "${OPENAI_API_KEY}"

  mcp:
    server_name: "alma-memory"
    mode: "stdio"

  retrieval:
    default_top_k: 10
    similarity_threshold: 0.7

  learning:
    min_occurrences_for_heuristic: 3
    confidence_decay_days: 30.0
```

Use `${ENV_VAR}` syntax for secrets -- never write actual keys to the config file.

### 3. Generate SQL Migration Script

For PostgreSQL+pgvector users, generate a migration script that creates the ALMA tables:

```sql
-- ALMA Open Brain Migration
-- Generated by: alma init --open-brain
-- Run: psql $DATABASE_URL -f .alma/migration.sql

-- Enable pgvector extension
CREATE EXTENSION IF NOT EXISTS vector;

-- ALMA tables (follow alma/storage/postgresql.py schema)
-- ... (tables for heuristics, outcomes, embeddings, metadata)
```

Reference `alma/storage/postgresql.py` for the exact table schema. The migration must be idempotent (`CREATE TABLE IF NOT EXISTS`).

### 4. Generate .mcp.json for Claude Code

```json
{

      "args": ["-m", "alma.mcp"],

  }

```

## Implementation Notes

### CLI Framework

Use Python's `argparse` for the CLI (no new dependencies). ALMA does not currently have a CLI module, so this creates `alma/cli/`:

```
alma/cli/
  __init__.py
  init.py        -- alma init command
  templates/
    config.yaml  -- Config template
    migration.sql -- SQL migration template
    mcp.json     -- MCP config template
```

Entry point in `pyproject.toml`:
```toml
[project.scripts]
alma = "alma.cli:main"
```

### Config Loader Compatibility

The generated YAML must work with `alma.config.loader.ConfigLoader.load()`. Key requirements:
- Root key must be `alma`
- Environment variable syntax: `${ENV_VAR}`
- KeyVault syntax (optional): `${KEYVAULT:secret_name}`

### Storage Backend Templates

Each backend needs a different config section:

- **SQLite+FAISS**: `path: ".alma/data"` (local directory)
- **PostgreSQL+pgvector**: `connection_string: "${ALMA_DATABASE_URL}"`
- **File-based**: `directory: ".alma/memories"` (local directory)
