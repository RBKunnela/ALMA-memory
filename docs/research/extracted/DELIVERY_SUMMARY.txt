================================================================================
WHAT COULD GO WRONG (AND HOW TO FIX IT)
================================================================================

Risk 1: Scoping adds latency that breaks retrieval accuracy
- Mitigation: Index on (agent_id, memory_type)
- Fallback: Remove scoping from retrieval, only in learning

Risk 2: Multi-agent queries are expensive
- Mitigation: Pre-cache sharing graphs at agent init time
- Fallback: Disable multi-agent sharing in MVP, add v1.1

Risk 3: Migration loses some MemPalace data
- Mitigation: Backup MemPalace data first
- Mitigation: Test migration on small dataset first
- Fallback: Keep MemPalace's ChromaDB as read-only fallback

Risk 4: PostgreSQL backend takes longer than Sprint 2
- Mitigation: Keep Chroma/SQLite as default, PostgreSQL optional
- Reality: You don't need it for MVP

Risk 5: You redesign the schema halfway through
- Mitigation: Commit to the SQL schema in week1_execution_guide.md
- Mitigation: If you want changes, document them in WEEK1_COMPLETE.md
